# 보안 점검 체크리스트

강의 자료(테스트 단계)에서 제시된 체크리스트 예시를 기준으로, 구현된 각
기능이 보안 요소를 제대로 만족하는지 항목별로 점검했다. `상태` 열은 최종
점검 결과이며, 자동 점검이 가능한 항목은 `tests/security_test.py` 로 검증했다.

## 1. 회원가입 및 프로필 관리

| 점검 항목 | 설명 | 구현 위치 | 상태 |
|---|---|---|---|
| 서버측 입력 검증 | 아이디/비밀번호 길이·문자 집합·형식을 서버에서 검증 | `security.validate_username/validate_password` | ✅ |
| CSRF 보호 | 모든 상태 변경 POST 에 CSRF 토큰 검증 | `extensions.csrf` (Flask-WTF, 전역) | ✅ |
| 비밀번호 보안 | 평문 저장 금지, bcrypt(자동 salt)로 해시 저장 | `security.hash_password` | ✅ |
| 세션 쿠키 설정 | HttpOnly, SameSite=Lax, HTTPS 시 Secure | `config.Config` 세션 옵션 | ✅ |
| 세션 만료 및 재인증 | 2시간 후 세션 만료, 비번 변경 시 현재 비번 재확인 | `config`, `auth.mypage` | ✅ |
| 실패 로그인 방어 | 5회 연속 실패 시 5분 계정 잠금 + 요청 속도 제한 | `auth._register_login_failure`, `limiter` | ✅ |
| 오류 메시지 | "아이디 없음/비번 틀림"을 구분하지 않아 계정 존재 여부 비노출 | `auth.login` | ✅ |
| 아이디 중복 방지 | DB UNIQUE 제약 + 가입 시 사전 확인 | `schema.sql`, `auth.register` | ✅ |

## 2. 상품 등록 및 관리

| 점검 항목 | 설명 | 구현 위치 | 상태 |
|---|---|---|---|
| 폼 입력 검증 | 상품명/설명/가격 필수값·길이·숫자·범위 검증 | `products.create/edit` | ✅ |
| XSS 방어 | 출력 시 Jinja2 자동 이스케이프 + CSP 헤더 | 템플릿 전반, `app.set_security_headers` | ✅ |
| 인증된 사용자만 등록 | 로그인 사용자만 상품 등록/수정/삭제 | `@login_required` | ✅ |
| 소유자 확인 | 상품 수정·삭제 시 소유자(또는 관리자)인지 검증 | `products.edit/delete` | ✅ |
| 파일 업로드 검증 | 확장자 화이트리스트 + Pillow 실제 이미지 검증 + 파일명 UUID 재생성 | `security.safe_image_save` | ✅ |
| 다중 이미지 업로드 | 장수(최대 6장)·전체 용량 상한, 각 파일 개별 이미지 검증 | `products._save_images`, `config` | ✅ |
| SQL Injection 방어 | 모든 쿼리 파라미터 바인딩, LIKE 특수문자 이스케이프 | `db.query/execute`, `products.index` | ✅ |

## 3. 실시간 채팅 및 메시징

| 점검 항목 | 설명 | 구현 위치 | 상태 |
|---|---|---|---|
| 메시지 내용 검증 | 빈 문자열/최대 길이(500자) 검증 | `events.on_send_message` | ✅ |
| 사용자 인증 | 소켓 세션에 로그인 정보 없으면 메시지 무시 | `events._authed_user` | ✅ |
| 1:1 방 접근 통제 | 방 이름에 본인 id 가 포함될 때만 join/전송 허용 | `events.on_join/on_send_message` | ✅ |
| Rate Limiting | 동일 사용자 0.5초 미만 연속 전송 차단(도배 방지) | `events._rate_ok` | ✅ |
| 전송 구간 암호화 | 운영(HTTPS/ngrok)에서 자동 wss 사용 | `chat.js`(io()), CSP `connect-src` | ✅ |
| 사진 전송 검증 | 업로드 전용 경로에서 이미지 검증·UUID 저장, 소켓은 검증된 파일명만 수신하고 형식(32자리+확장자)·존재 여부 재확인(경로 조작 차단) | `community.chat_upload`, `events.on_send_message`, `security.is_valid_stored_image` | ✅ |
