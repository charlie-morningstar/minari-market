## 2. 수동 점검 (기능 동작 확인)

두 개 이상의 브라우저(또는 PC + 휴대폰, ngrok 사용)로 접속하여 아래 시나리오를
직접 확인했다.

| 기능 | 시나리오 | 결과 |
|---|---|---|
| 회원가입/로그인 | 규칙 위반 입력 시 오류 안내, 정상 입력 시 가입/로그인 | 정상 |
| 상품 등록/조회 | 이미지 포함 등록 → 목록·상세 노출 | 정상 |

| 전체 채팅 | 두 브라우저에서 실시간 메시지 송수신 | 정상 |
| 1:1 쪽지 | 상대 프로필/상품에서 대화 시작, 당사자만 열람 | 정상 |

## 3. 알려진 한계 / 후속 과제

- 현재 채팅 도배 방지는 서버 프로세스 메모리 기반(단일 인스턴스 가정)이다.
  다중 인스턴스로 확장한다면 Redis 등 공용 저장소 기반으로 옮겨야 한다.
- 실시간 전체 채팅은 최근 50개 메시지만 로드한다(무한 스크롤 미구현).
- 송금은 가상 포인트이며 실제 결제(PG) 연동은 과제 범위에서 제외했다.
