# 미나리마켓

동네 중고거래 사이트를 만들어 봤다. 화이트햇스쿨 시큐어코딩 과제인데, 기능
돌아가게 하는 것보다 "안 털리게" 만드는 데 더 신경 썼다. 폰이나 IT 기기 중고로
사고파는 걸 생각하고 만들었다.

맥(파이썬 3.10 이상)에서 만들고 돌려봤다. 리눅스/WSL에서도 똑같이 된다.

## 뭐가 되냐

- 회원가입 / 로그인 / 마이페이지(소개글·비번 바꾸기) / 남 프로필 보기
- 상품 등록(사진 여러 장) / 목록·상세·검색 / 내가 올린 거 수정·삭제
- 전체 채팅이랑 1:1 쪽지. 실시간이고 사진도 보낼 수 있다
- 상품이나 사람 신고 → 쌓이면 자동으로 상품 숨기거나 계정 휴면
- 포인트 충전해서 사람끼리 송금
- 관리자 페이지(사용자·상품·신고 관리)

## 돌리는 법

이거 한 줄이면 된다. 가상환경 만들고, 필요한 거 깔고, DB 만들고, 서버까지 켜 준다.

```bash
./run.sh
```

그리고 브라우저에서 `localhost:5001` 열면 된다.

직접 하나씩 하고 싶으면:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env      # 그리고 SECRET_KEY 채우기 (아래 참고)
python init_db.py
python app.py
```

`.env`의 `SECRET_KEY`는 이 명령 출력값을 넣으면 된다:

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

포트는 5001이다. 맥은 5000번을 에어플레이가 쓰고 있어서 바꿔놨다.

처음 만들어지는 계정:

## 폰으로도 채팅되나 보고 싶으면

서버 켜 둔 채로 다른 터미널에서:

```bash
ngrok http 5001
```

나오는 `https://....ngrok-free.app` 주소를 폰에서 열면 된다. (ngrok 깔려 있어야 함.
없으면 그냥 컴퓨터에서만 봐도 과제 확인엔 지장 없다.)

## 보안 잘 막혔나 확인

서버 켜 놓고 다른 터미널에서:

```bash
pip install requests     # 한 번만
python tests/security_test.py
```

CSRF, XSS, SQL 인젝션, 로그인/권한, 송금, 사진 업로드 같은 걸 실제로 요청 날려서
확인한다. 다 통과하면 몇 개 통과했다고 찍힌다.

## 폴더 구조

```
app.py            서버 켜는 진입점 (+보안 헤더, 오류 페이지)
config.py         설정값 (세션/쿠키/업로드 제한/정책 숫자)
db.py             DB 연결이랑 쿼리 (값은 전부 ? 로 바인딩)
security.py       해시, 입력 검증, 로그인/권한 체크, 업로드 검증
events.py         실시간 채팅(소켓) 처리
schema.sql        테이블 정의
init_db.py        DB 초기화 + 기본 계정/샘플 상품
blueprints/       회원 · 상품 · 커뮤니티(채팅/신고) · 지갑 · 관리자
templates/        화면(HTML)
static/           css, 채팅/갤러리 js, 업로드된 사진
tests/            보안 자동 점검
docs/             체크리스트, 테스트 결과, 보고서, 설계 그림
```

## 보안은 이렇게 챙겼다

자세한 건 보고서에 적어놨고, 짧게 요약하면:

- 비번은 bcrypt로만 저장 (평문 저장 안 함)
- 사용자가 보낸 값은 서버에서 다시 검사 (허용된 형태만 통과)
- SQL은 전부 `?` 바인딩, 검색어 특수문자도 처리
- 화면 출력은 자동 이스케이프, 실시간 채팅은 textContent로만 (innerHTML 안 씀) + CSP
- 상태 바꾸는 요청엔 CSRF 토큰
- 로그인/소유자/관리자 권한 체크
- 사진 업로드는 진짜 이미지인지 열어서 확인하고 파일명 새로 지어 저장,
  채팅 사진은 검증된 파일명만 오가게 해서 경로 조작 막음
- 송금은 잔액·금액 확인하고 하나의 트랜잭션으로 (중간에 틀어지면 되돌림)
- 로그인 실패 잠금, 요청 속도 제한, 오류 시 내부정보 안 보이게, 비밀키는 .env로 분리
